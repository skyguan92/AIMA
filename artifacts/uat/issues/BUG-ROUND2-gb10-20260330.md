# Round-2 gb10 Bug Report

**机器**：`gb10`  
**日期**：2026-03-30  
**构建**：`v0.2-dev` / commit `77f815a`

## BUG-R2-004：`aima run` 的大模型自动下载恢复能力不足，长时间无法进入部署

- **场景**：空目录下运行 `qwen3-4b`
- **期望**：引擎确认后，模型下载即使遇到网络抖动也能稳定恢复，并继续进入部署
- **实际**：
  - `aima run` 成功进入模型自动下载
  - Hugging Face 下载阶段多次出现超时、SSL EOF、`ChunkedEncodingError`
  - AIMA 尝试切换到 ModelScope
  - 切源后又进入一轮新的大下载，整条链路在 15 分钟以上仍未 ready
- **影响**：高。一键运行无法在真实不稳定网络下完成。
- **用户视角问题**：用户看到的是“已经自动开始了”，但很久都没有服务可用，也没有稳定的完成预期。

## BUG-R2-005：切源后的续传感知弱，用户体感接近重新下载

- **场景**：Hugging Face 大文件下载中途失败后自动切到 ModelScope
- **期望**：已完成的数据尽量被复用，或者至少给用户明确说明“正在断点续传”
- **实际**：
  - 日志显示进入新的多文件下载过程
  - 用户很难判断这是不是在复用已有数据，还是重新来过
  - 真实体验更像“换了一个源重新下”
- **影响**：中到高。即使底层技术上有恢复，用户体验仍然像链路失控。

## NOTICE-R2-001：Hugging Face cache 权限告警会污染一键运行体验

- **现象**：下载开始时出现 `Permission denied` cache 告警
- **影响**：低到中。未立即中断，但会显著降低用户对自动化链路的信任。
