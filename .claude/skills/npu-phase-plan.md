## GEMM 测试失败根本原因分析（2026-02-27）

### 问题诊断

通过代码分析和测试验证，发现 GEMM 测试失败的根本原因：

| 层面 | 状态 | 说明 |
|--------|------|------|
| NPU5/Strix Halo 硬件识别 | ✅ 正常 | AIE2P, 6x8 topology |
| Development Firmware | ✅ 已加载 | 255.0.11.71 |
| Latency 测试 | ✅ PASSED | 62 μs |
| Throughput 测试 | ✅ PASSED | 63,675 op/s |
| **GEMM 测试** | ❌ **FAILED** | cycle count is 0 |

### 根本原因：测试文件不兼容 NPU5

所有可用的 xclbin 和 ELF 文件都是为**旧硬件设计的**：

```
xclbin 信息：
  Platform: xilinx_v1_ipu_0_0
  Generated by: v++ 2021.2.0 (2021年)
  Target: IPUv1 (AIE2PS) ← 不是 AIE2P
  Kernel: vadd (向量加法) ← 不是 GEMM

当前设备：
  Platform: NPU5/Strix Halo (AIE2P)
  Device ID: 0x17f0
  需要: AIE2P 兼容的 xclbin/ELF 文件
```

### 驱动层发现

```cpp
// dev_npu5_info 结构定义
const struct amdxdna_dev_info dev_npu5_info = {
    .default_vbnv = "NPU Strix Halo",
    .dev_priv = &npu5_dev_priv,
    NPU4_COMMON_DEV_INFO,
};

const struct amdxdna_dev_priv npu5_dev_priv = {
    .fw_path = "amdnpu/17f0_11/npu.sbin",  // Production
    .min_fw_version = 0,  // 跳过版本检查（支持 development firmware）
    NPU4_COMMON_DEV_PRIV,
};
```

**结论**：驱动正确识别并处理 NPU5 设备，但测试套件中没有为 NPU5 准备的测试文件。

### 工具链问题

| 组件 | 状态 |
|--------|------|
| AIE2 编译器 | ❌ 缺失（MLIR-AIE AIE2 backend） |
| GEMM xclbin 编译器 | ❌ 缺失（v++ 不支持 NPU5） |
| NPU5 专用测试文件 | ❌ 不存在 |

### 解决方案评估

| 方案 | 可行性 | 说明 |
|------|----------|------|
| 使用现有 xclbin | ❌ | 文件不兼容 NPU5 |
| 重新编译 GEMM xclbin | ❌ | 缺少 AIE2 编译工具链 |
| 修改 xclbin 以兼容 | ❌ | 二进制格式复杂，不可行 |
| 等待官方 AMD SDK | ✅ | 2026 Q3-Q4 预期发布 |

### 最终结论

**GEMM 测试失败不是固件或硬件问题，而是完整的工具链问题。**

- Development Firmware 255.0.11.71 改善了基础 NPU 响应性
- 但没有为 NPU5/AIE2P 准备的 GEMM compute kernel 文件
- 需要官方 AMD Ryzen AI Linux SDK 提供完整的 AIE2P 工具链

**建议**：将 NPU 作为预留选项，当前使用 Vulkan iGPU 作为推理后端。

## GEMM 解决方案验证（2026-02-28）

### 问题根源确认

**xrt-smi validate GEMM 测试失败**：
```
Device           : NPU Strix Halo
Platform         : AIE2P (6x8 topology)
Cycle Count      : 0
Test Status      : FAILED
```

**根本原因**：
`/tmp/gemm.xclbin` 文件是为 AIE2PS (NPU3) 设备编译的，不兼容 NPU5/AIE2P。

### MLIR-AIE 编译尝试结果

**结论**：
当前环境（Ubuntu 24.04, Python 外部管理）无法完成 MLIR-AIE 从源码编译。

### 官方解决方案发现

**来源**：Xilinx/mlir-aie GitHub 仓库 + AMD Ryzen AI SW

**关键文件**：
- `programming_examples/basic/matrix_multiplication/single_core/single_core_iron.py`
- 支持 NPU2 (AIE2P/XDNA2) 设备

**问题**：
- AMD Ryzen AI SW 仓库包含完整的 NPU 推理流水线示例
- 但没有独立的、可直接使用的 GEMM xclbin 预编译文件

### 验证结论

| 检查项 | 结果 |
|--------|------|
| NPU 设备识别 | ✅ NPU Strix Halo, AIE2P, 6x8 |
| Development Firmware | ✅ 255.0.11.71 已加载 |
| Latency Test | ✅ PASSED (62 μs) |
| Throughput Test | ✅ PASSED (65,961 op/s) |
| GEMM Test | ❌ FAILED (cycle count 0) |

**最终结论**：

1. **GEMM 测试失败不是硬件或 firmware 问题**
2. **根本原因是测试文件架构不匹配**：
   - 当前使用的 `/tmp/gemm.xclbin` 是为 AIE2PS (NPU3) 设计的
   - NPU5/AIE2P (Strix Halo) 需要不同架构的测试文件

4. **可行的解决方案路径**：
   - 等待 AMD Ryzen AI Linux SDK (预计 2026 Q3-Q4) 提供完整 AIE2P 工具链
   - 或使用 Windows + Ryzen AI SDK (GAIA 项目已支持 NPU)
   - 当前生产路径：Vulkan iGPU (已验证 PP512: 1032 tok/s)
