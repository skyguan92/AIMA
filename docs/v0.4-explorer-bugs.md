# v0.4 Explorer 测试 Bug 清单

日期: 2026-04-07
测试设备: linux-1 (cjwx)

---

## 测试中发现并修复

| # | 严重度 | 步骤 | 描述 | 修复状态 |
|---|--------|------|------|----------|
| B2 | **P1** | Step 3 | OpenAICompleter 未设置 User-Agent header, Kimi 返回 403 `access_terminated_error` | ✅ 已修 — 新增 `WithOpenAIHeaders` option + `CENTRAL_LLM_HEADERS` 环境变量 |
| B3 | **P1** | Step 4 | Advisor 解析 LLM 响应时剥离了 markdown code fences, 但存储到 `ContentJSON`/`Details` 的仍是原始 raw (含 fences), 导致 `writeJSON` Marshal 失败返回 `internal error` | ✅ 已修 — `stripCodeFence` 统一在 `raw` 变量层面处理, Analyzer 同步修复 |
| B4 | **P2** | Step 5 | Edge syncHTTPClient timeout 30s, 对 LLM 调用不够 (Kimi 响应 10-30s), 导致 `context deadline exceeded` | ✅ 已修 — 改为 120s |
| B5 | **P2** | Step 8 | Scenario YAML 的 deployments 字段序列化为字节数组, 因 `[]json.RawMessage` 被 yaml.Marshal 当 `[][]byte` 处理 | ✅ 已修 — 先 Unmarshal 为 `[]any` 再传 yaml.Marshal |
| B6 | **P2** | Step 11 | LLMPlanner 生成 `compare` kind 任务, 但 ExplorationManager 不支持此 kind, 返回 `exploration kind "compare" not implemented` | ✅ 已修 — 从 LLMPlanner system prompt 中移除 `compare`, 明确约束可用 kind 为 validate/tune/open_question |
| B7 | **P2** | Step 10+11 | Auto-promote 不检查 throughput, 0 tok/s 的 config 仍被提升为 golden。在无实际推理服务时, 会产生大量无意义的 "golden" 配置 | ✅ 已修 — `maybeAutoPromote` 前置 `newThroughput > 0` 检查, 零吞吐不提升 |
| B8 | **P3** | Step 11 | LLM plan 生成耗时约 2 分钟。PlanInput 包含 243 个 gaps + 大量 open questions, 序列化后 prompt 过长 | ✅ 已修 — LLMPlanner 新增 `filterPlanInput`: 仅发送本机硬件 gaps (max 20), open questions (max 10), history (max 5) |

## 设计观察 (已修)

| # | 描述 | 修复 |
|---|------|------|
| O4 | Explorer 无法自行升级 Tier 1→2: `detectTier()` 依赖 `Agent.ToolMode()`, 而 ToolMode 仅在 `Agent.Ask()` 中检测, LLMPlanner 直接调 `llm.ChatCompletion()` 绕过检测 | ✅ 已修 — `refreshTier()` 中主动调 `Agent.ProbeToolMode()` 做轻量级 tool mode 检测 |
| O5 | CLI 模式 (`aima explorer trigger`) 是 fire-and-forget: 新进程启动 → 发事件 → 进程退出, 后台 goroutine 来不及处理 | ✅ 已修 — CLI trigger 增加 Long description 说明需要 serve 模式, 引导用户使用 MCP |
| O6 | Kimi LLMPlanner 生成的 plan 中 `Reasoning` 字段包含原始 LLM 响应 (含 markdown code fences), 而非清理后的文本 | ✅ 已修 — `parsePlanResponse` 对 Reasoning 也使用 stripped 版本 |

## 设计观察 (非 Bug, 保留关注)

| # | 描述 | 影响 |
|---|------|------|
| O1 | Central endpoint 配置存 SQLite 而非环境变量, 首次需 `aima config set central.endpoint <url>` | 增加了初始化步骤, 但好处是持久化 |
| O2 | Advisory 拉取时自动 pending→delivered 状态转换, 即使 edge 还没实际验证 | 符合设计意图 (delivered = 已送达 edge, validated = edge 确认) |
